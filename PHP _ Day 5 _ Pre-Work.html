JSON & PHP

A common use of JSON is to read data from a web server, and display the data in a web page. This section will cover how to exchange JSON data between the client and a PHP server.

PHP - Server Side

PHP has some built-in functions to handle JSON. Objects in PHP can be converted into JSON by using the PHP function json_encode(), like this:

<?php

if (!isset($myObj)) $myObj = new stdClass(); // check whether the object already exists

    $myObj->name = "John";

    $myObj->age = 30;

    $myObj->city = "New York";

    $myJSON = json_encode($myObj);

    echo $myJSON;

/* The output is JSON object

    { "name":"John", "age":30,"city": "New York" }

*/

?>

Save the code above in a php file called server-side.php. The example converts PHP object into JSON object. While executing, it produces the following output:

{ "name":"John", "age":30,"city": "New York" }

Javascript - Client Side

Since the output from the PHP file is a JSON object, JavaScript can access it with ease. On the client side (web browser) we use JavaScript, but to make the interaction between the server and the client smoother we will use an AJAX call. Create a new file and name it index.php:

<!DOCTYPE html>

<html>

<body>

<h2>Get data as JSON from a PHP file on the server.</h2>

<p id="placeholder"></p>

<script>

        var xmlhttp = new XMLHttpRequest();

        xmlhttp.onreadystatechange = function() {

                if (this.readyState == 4 && this.status== 200) {

                    myObj = JSON.parse(this.responseText);

                    text = 'My name is ' + myObj.name + ' and I am from ' + myObj.city;

                    document.getElementById("placeholder").innerHTML= text;

                }

        };

        xmlhttp.open("GET", "server-side.php", true);

        xmlhttp.send();

</script>

</body>

</html>

The output will be:

My name is John and I am from New York

We are accessing the properties name and city from the myObj object, store them in a variable called text and display it within the paragraph with the id of  “placeholder”.

Database & PHP & Javascript using GET Method

PHP is a server side programming language, and should be used for operations that can only be performed by a server, like accessing a database.

Built-in conversion Methods

Both PHP and Javascript have built-in methods for dealing with object/JSON conversion. In most cases we use the JSON format to pass data and Object to access specific properties from an object.

PHP

json_encode() - converts Object into JSON

json_decode() - converts JSON into Object

Javascript

json.stringify() - converts Object into JSON

json.parse() - converts JSON into Object

In the following example we will pass the parameters that should be retrieved from the database using the GET method. Since we use AJAX the parameters will be not display in the URL, instead AJAX works behind the scene. But if we open the console we can see how the data is passed:

Database

Create a database called “google” and create within it one table called “employees”:

Insert the following data in the database:

INSERT INTO `employees` (`name`, `position`, `salary`) VALUES ('Mike', 'android developer', '94000'), ('John', 'software engineer', '78000'), ('Steven', 'data analyst', '85000'), ('Carol', 'software engineer', '94000'), ('Jim', 'android developer', '83000'), ('Robert', 'data analyst', '79000'), ('Bill', 'project manager', '112000'), ('Georg', 'software engineer', '93000'), ('Boris', 'android developer', '97000'), ('Brian', 'data analyst', '82000');

We want to make a request to the server where we ask for the salaries greater than 90000 in the "employees" table.

PHP - Server Side

On the server side we have the following code:

<?php

        header("Content-Type: application/json; charset=UTF-8");

        $obj = json_decode($_GET["x"], false); // Convert the request into an object

        $conn = new mysqli("localhost", "root", "", "google"); // Access the database uising mysqli

        $result = $conn->query("SELECT name FROM " . $obj->table . " WHERE position = " . $obj->position);

        $outp = array(); // create an empty array

        $outp = $result->fetch_all(MYSQLI_ASSOC); // fill an array with the requested data and store it in the array

        echo json_encode($outp); // return the object as JSON

?>

The query:

$result = $conn->query("SELECT name FROM " . $obj->table . " WHERE position = " . $obj->position);

is same as the following one:

SELECT * FROM `employees` WHERE `position` = 'android developer'

We use the first approach because we want to retrieve the data based on the GET parameters. In this case the name of the table and position were passed.

Javascript - Client Side

On the client side we have the following code:

<!DOCTYPE html>

<html>

<body>

<h2>Get data as JSON from a PHP file on the server.</h2>

<p>The JSON received from the PHP file:</p>

<p id="demo"></p>

<script>

        var obj, dbParam, xmlhttp;

        obj = { "table":"employees", "position":"'android developer'" }; // Define an object containing a table property and a limit property

        dbParam = JSON.stringify(obj); // Convert the Object into a JSON

        xmlhttp = new XMLHttpRequest();

        xmlhttp.onreadystatechange = function() {

                if (this.readyState == 4 && this.status== 200) {

                    document.getElementById("demo").innerHTML= this.responseText;

                }

        };

        xmlhttp.open("GET", "server-side.php?x=" + dbParam, true); // Send a request to the PHP file, with the JSON string as a parameters

        xmlhttp.send(); // Display the result received from the PHP file

</script>

</body>

</html>

It returns the following result from the database:

[{"name":"Mike"},{"name":"Jim"},{"name":"Boris"}]

Loop through the results

For Loop through the properties of an object we would use for/in Statement. The for/in statement loops through the properties of an object. The block of code inside the loop will be executed once for each property.

<!DOCTYPE html>

<html>

<body>

<h2>Get data as JSON from a PHP file on the server.</h2>

<p>The JSON received from the PHP file:</p>

<p id="demo"></p>

<script>

        var obj, dbParam, xmlhttp, myObj, i, store="";

        obj = { "table":"employees", "position":"'android developer'" }; // Define an object containing a table property and a limit property

        dbParam = JSON.stringify(obj);

        xmlhttp = new XMLHttpRequest();

        xmlhttp.onreadystatechange = function() {

                if (this.readyState == 4 && this.status== 200) {

                    myObj = JSON.parse(this.responseText);

                    for (i in myObj) {

                        store += myObj[i].name + "<br>";

                    }

                    document.getElementById("demo").innerHTML = store;

                }

        };

        xmlhttp.open("GET", "server-side.php?x=" + dbParam, true);

        xmlhttp.send();

</script>

</body>

</html>

Within the for/in statement, i is a variable that iterates over the properties of the object

myObj. -  is the specified object that will be iterated.

Database & PHP & Javascript using POST Method

When sending data to the server, it is often best to use the HTTP POST method. To send AJAX requests using the POST method, we need to specify the method and the correct header.

The data sent to the server must now be an argument to the .send() method:

<!DOCTYPE html>

<html>

<body>

<h2>Get data as JSON from a PHP file on the server.</h2>

<p>The JSON received from the PHP file:</p>

<p id="demo"></p>

<script>

        var obj, dbParam, xmlhttp, myObj, i, store="";

        obj = { "table":"employees", "position":"'android developer'" }; // Define an object containing a table property and a limit property

        dbParam = JSON.stringify(obj);

        xmlhttp = new XMLHttpRequest();

        xmlhttp.onreadystatechange = function() {

                if (this.readyState == 4 && this.status== 200) {

                    myObj = JSON.parse(this.responseText);

                    for (i in myObj) {

                        store += myObj[i].name + "<br>";

                    }

                    document.getElementById("demo").innerHTML = store;

                }

        };

        xmlhttp.open("POST", "server-side.php", true)

        xmlhttp.setRequestHeader("Content-type", "application/x-www-form-urlencoded");

        xmlhttp.send("x=" + dbParam);

</script>

</body>

</html>

The only difference in the PHP file is the method for getting the transferred data. Use $_POST instead of $_GET:

<?php

        header("Content-Type: application/json; charset=UTF-8");

        $obj = json_decode($_POST["x"], false); // Convert the request from the GET into an object

        $conn = new mysqli("localhost", "root", "", "google"); // Access the database using mysqli

        $result = $conn->query("SELECT name FROM " . $obj->table . " WHERE position = " . $obj->position);

        $outp = array(); // create an empty array

        $outp = $result->fetch_all(MYSQLI_ASSOC); // fill an array with the requested data an store it in the array

        echo json_encode($outp); // return the object as JSON

?>

Insert Data into the Database

Version 1 - Without AJAX

First we will create a simple form, which contains tree input fields: name, salary and position. This form will use the POST method and will be further processed by the server-side.php file:

<!DOCTYPE html>

<html>

<head>

        <title>Insert Data into MySQL</title>

</head>

<body>

        <form action="server-side.php" method="POST">

                <input type="text" name="name" placeholder="name">

                <input type="text" name="position" placeholder="position">

                <input type="text" name="salary" placeholder="salary">

                <input type="submit" id="submit" id="sub">

        </form>

        <span id="message"></span>

</body>

</html>

Once the form is submitted we need to fetch the submitted data and work on it. Let’s code the back-end. Create a new file called server-side.php and write the following code inside of it.

<?php

        $conn = new mysqli("localhost", "root", "", "google");

        $name = $_POST['name'];

        $salary = $_POST['salary'];

        $position = $_POST['position'];

        if ($conn->query("INSERT INTO employees (name, salary, position) VALUES ('$name', '$salary', '$position')")) {

                 echo "Successfully Inserted new Employee";

         } else {

                 echo "Unsuccessful Insertion";

         }

?>

Version 2 - With AJAX

If we want to stay on the same page without reloading it and immediately get the response instead of being redirected after a successful Insert query,  we can use AJAX with JQuery.

Create a new Javascript file called script-ajax.js

$("#sub").click( function() { // when submit button is clicked
$.post( $("#myForm").attr("action"), // fetch the URL from myForm
        $("#myForm :input").serializeArray(), // serialize the data from the input fields, creating an array of objects (name and value pairs)
        function(info){ $("#message").html(info); // append the response to element with id of message
  });
clearInput(); // Call the customized clearInput()
});

$("#myForm").submit( function() {
 return false;        // do not redirect after submit is clicked
});

function clearInput() { // clear all input fields from #myForm
       $("#myForm :input").each( function() {
              $(this).val('');
       });
}

Once the user clicks on the button with the id “sub”, $.post() method is invoked. $.post() is a shortcut to the $.ajax() method. The general syntax is:

 $.post( 'pass_data_to_this_url', 'data', 'callback_function' );

  • We fetch the URL from the myForm form, from its action attribute.
  • Serialize all the user entered inputs.
  • SerializeArray makes the input into “property: value” pair.
  • After the data is passed to the url and is being processed, the result will be returned back to the callback function and is kept  in the info variable. It is inserted into the span tag using the html() method – which is similar to the innerHTML() method of JavaScript.
  • Disable Form Redirection. We select the form and make sure to return false upon submission.
  • We wrote a custom function called clearInput() which selects the input fields and sets each of its values to none or empty.

Using the AJAX technique, we would make sure that the form doesn’t redirect to the server-side.php file after the submission, and we’ll display the appropriate message to the user on index.php file itself after the user entered values have been processed by the server-side.php file.

Link this file to the index.php adding these two lines before the closing body tag:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.2.1/jquery.min.js"></script>
<script src="script-ajax.js" type="text/javascript"></script>
